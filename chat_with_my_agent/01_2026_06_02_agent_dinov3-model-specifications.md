## 一、DINOv3 模型總覽（共 10 個模型）

DINOv3 分為 **Vision Transformer (ViT)** 與 **ConvNeXt** 兩大架構。

### Vision Transformer 模型

| 代號 | 全名 | 引數數量 | Embedding Dim | Attention Heads | FFN 型別 |
|------|------|----------|---------------|-----------------|----------|
| **S/16** | ViT-Small /16 | 21M | 384 | 6 | MLP |
| **S+/16** | ViT-Small+ /16 | 29M | 384 | 6 | SwiGLU |
| **B/16** | ViT-Base /16 | 86M | 768 | 12 | MLP |
| **L/16** | ViT-Large /16 | 300M | 1024 | 16 | MLP |
| **H+/16** | ViT-Huge+ /16 | 840M | 1280 | 20 | SwiGLU |
| **7B/16** | ViT-7B /16 | 6,716M | 4096 | 32 | SwiGLU |

> Python 驗證：
> - 引數數量：`21, 29, 86, 300, 840, 6716`（單位：M = millions）
> - Embedding Dim：`384, 768, 1024, 1280, 4096`
> - Attention Heads：`6, 12, 16, 20, 32`

### ConvNeXt 模型（皆由 ViT-7B 知識蒸餠得到）

| 代號 | 全名 | 引數數量 |
|------|------|----------|
| **T** | ConvNeXt-Tiny | 29M |
| **S** | ConvNeXt-Small | 50M |
| **B** | ConvNeXt-Base | 89M |
| **L** | ConvNeXt-Large | 198M |

> Python 驗證：ConvNeXt 引數為 `29, 50, 89, 198`（單位：M）
>
> 相對於同級 ViT 的比例：
> - Tiny(29M) vs ViT-S(21M)：`29/21 = 1.38x`
> - Small(50M) vs ViT-S+(29M)：`50/29 = 1.72x`
> - Base(89M) vs ViT-B(86M)：`89/86 = 1.03x`
> - Large(198M) vs ViT-L(300M)：`198/300 = 0.66x`

### 訓練來源分類

| 訓練方式 | 模型 |
|----------|------|
| **From Scratch**（原始訓練） | ViT-7B/16（網路資料 LVD-1689M）、ViT-7B/16（衛星資料 SAT-493M） |
| **Knowledge Distillation**（從 ViT-7B 蒸餠） | ViT-S, S+, B, L, H+、ConvNeXt-T/S/B/L、ViT-L（衛星資料） |

### 模型數量與訓練資料總合

|------|------|------|
| ViT 模型數 | 6 | |
| ConvNeXt 模型數 | 4 | |

## 二、代號含意

### 字母代號（模型大小等級）

| 代號 | 含意 | 說明 | 比例驗證 |
|------|------|------|----------|
| **S** | Small | 小型模型，適合資源受限或輕量應用 | 基準 |
| **S+** | Small+ | Small 的增強版，改用 SwiGLU FFN 替代 MLP，引數略增（21M → 29M） | `29 / 21 = 1.38x` |
| **B** | Base | 基準模型，平衡效能與成本 | `86 / 21 = 4.10x` |
| **L** | Large | 大型模型，提供更高品質特徵 | `300 / 21 = 14.29x` |
| **H+** | Huge+ | 超大型模型的增強版，改用 SwiGLU FFN | `840 / 21 = 40.00x` |
| **7B** | 7 Billion | 教師模型（Teacher），6,716M 引數，用於蒸餠與研究 | `6716 / 21 = 319.81x ≈ 320x` |
| **T** | Tiny | ConvNeXt 架構的最小模型 | — |

> ⚠️ DINOv3 **沒有** `n`（Nano）、`g`（Giant）或 `h14/14` 這些代號。  
> `h14/14` 與 `g/14` 是 **DINOv2** 的尺寸。

### 數字（如 16）

| 數字 | 含意 |
|------|------|
| **16** | 代表 **patch size**，即輸入影像被切成 16×16 畫素的影像塊（patch / token） |

所有 ViT 模型的 patch size 統一為 **16**。ConvNeXt 模型不使用此表示法，因為它是卷積架構。

## 三、區域特徵對應原始畫素

### 基本對應關係

所有 ViT 模型的 patch size 都是 **16×16**，因此：

| 專案 | 數值 | 驗證 |
|------|------|------|
| 每個 Patch 對應畫素 | **256 畫素** | `16 * 16 = 256` |
| 224×224 輸入的 Patch 網格 | 224 / 16 = 14，14 × 14 = **196 個 patch token** | `224 / 16 = 14`，`14 * 14 = 196` |

### Token 組成（224×224 輸入為例）

```
Class Token     : 1 個   （全域性特徵）
Register Tokens : 4 個   （用於降低 high-norm artifact，改善密集預測）
Patch Tokens    : 196 個 （14×14 網格，每個對應 16×16 = 256 畫素）
──────────────────────────────────────────────────────────────
Total Tokens    : **201 個** （python: 1 + 4 + 196 = 201）
```

### 各模型詳細對應

| 模型 | Patch 尺寸 | 每個 Patch 對應畫素 | 224×224 → Patch 數量 |
|------|-----------|-------------------|---------------------|
| ViT-S/16 | 16×16 | **256 畫素** | 196 |
| ViT-S+/16 | 16×16 | **256 畫素** | 196 |
| ViT-B/16 | 16×16 | **256 畫素** | 196 |
| ViT-L/16 | 16×16 | **256 畫素** | 196 |
| ViT-H+/16 | 16×16 | **256 畫素** | 196 |
| ViT-7B/16 | 16×16 | **256 畫素** | 196 |

> 所有 ViT 模型的 patch 對應關係相同，因為 patch size 全部為 16。

### 感受野（Receptive Field）

| 層級 | 感受野 |
|------|--------|
| 最淺層 | ≈ 16×16 = **256 畫素**（等於 patch size） |
| 最後一層 | ≈ **整個輸入影像**（Self-Attention 讓每個 token 關注所有 token） |

ConvNeXt 模型因使用卷積架構，感受野隨層數逐漸擴大，沒有 patch 概念的直接對應。

## 四、DINOv2 vs DINOv3 快速對照

| 方面 | DINOv2 | DINOv3 |
|------|--------|--------|
| patch size | 14（h14/g14）或 16（s/b/l） | **全部 16** |
| 最大模型 | ViT-g/14（~1.1B 引數） | ViT-7B/16（6,716M 引數） |
| 架構 | 僅 ViT | ViT + ConvNeXt |
| 模型數量 | ~3 個 ViT | 6 個 ViT + 4 個 ConvNeXt = **10 個** |
| 訓練資料 | 142M 影像 | 1.689B（網路）+ 493M（衛星） |

> Python 驗證：
> - 模型總數：`6 + 4 = 10`
> - 資料總和：`1689 + 493 = 2182M = 2.182B`
> - 最大模型：`6716 / 1000 = 6.716B ≈ 7B`
